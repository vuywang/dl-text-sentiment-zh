# 项目答辩演示说明

## 1. 项目一句话介绍

本项目是一个基于 BERT 的中文文本情感分析系统，可以对中文文本进行积极/消极二分类，并提供单文本分析、CSV 批量分析、模型训练、模型评估和历史记录管理。

答辩时可以这样开场：

```text
我的项目是一个基于深度学习的中文情感分析系统。它使用 bert-base-chinese 作为预训练模型，在 ChnSentiCorp 中文情感数据集上进行训练，并通过 FastAPI 做成可直接演示的 Web 系统。系统不只是一个模型脚本，还包含预测、批量处理、训练、评估和数据库持久化。
```

## 3. 推荐演示顺序

### 第一步：首页

打开：

```text
http://127.0.0.1:8000/
```

讲解重点：

- 这是一个完整 Web 系统，不是只跑一个 notebook。
- 首页可以看到项目名称、当前激活模型、数据库连接状态和最近记录。
- 数据库使用 SQLite，记录会保存在 `storage/db/app.db`。

可以这样说：

```text
首页主要用来展示系统整体状态，包括当前激活模型、数据库状态、最近训练记录和最近分析记录。这说明系统启动后已经完成了目录检查、数据库初始化和模型加载。
```

### 第二步：单文本情感分析

打开：

```text
http://127.0.0.1:8000/predict
```

建议输入积极样例：

```text
这家酒店服务很好，房间干净整洁。
```

建议输入消极样例：

```text
物流太慢了，包装也有破损。
```

讲解重点：

- 输入中文文本后，系统会先做文本校验。
- 后端加载当前激活模型，对文本做 tokenizer 编码。
- 模型输出两个类别的概率，系统取概率更高的一类作为预测标签。
- 预测结果会写入 `analysis_record` 表。

可以这样说：

```text
单文本预测流程是：中文输入先经过 tokenizer 转成模型能理解的 token，然后送入 BERT 分类模型，最后通过 softmax 得到积极和消极的概率。页面展示的是最终标签、置信度和两个类别的概率。
```

### 第三步：CSV 批量分析

打开：

```text
http://127.0.0.1:8000/batch
```

上传示例文件：

```text
storage/datasets/sample_batch.csv
```

讲解重点：

- CSV 必须包含 `text` 列。
- 系统会逐行预测，统计积极和消极数量。
- 结果文件会导出到 `storage/exports/`。
- 批量任务会写入 `batch_task` 表。

命令行也可以演示：

```powershell
python scripts/batch_predict.py --input storage/datasets/sample_batch.csv --output storage/exports/demo_result.csv
```

可以这样说：

```text
批量分析证明系统不只是单条文本预测，还可以处理 CSV 文件。后端会检查 text 列，逐行调用同一个预测服务，最后生成新的 CSV 结果并保存任务记录。
```

### 第四步：模型训练

打开：

```text
http://127.0.0.1:8000/train
```

讲解重点：

- 使用 `google-bert/bert-base-chinese` 作为预训练模型。
- 使用 ChnSentiCorp 作为中文情感分类数据集。
- 标签含义是 `0=消极`，`1=积极`。
- 页面提交训练参数后，后端会创建训练任务，再启动 `scripts/train_model.py` 子进程。
- 训练完成后保存模型、tokenizer、训练配置、评估指标、混淆矩阵和损失曲线。

现场不要完整跑长训练，可以说明：

```text
因为 BERT 训练耗时较长，答辩现场我主要展示训练入口和已经完成的训练结果。如果需要现场演示，可以把 epoch 设为 1，batch_size 设为 8，max_length 设为 64。
```

### 第五步：模型评估

打开：

```text
http://127.0.0.1:8000/evaluate
```

讲解重点：

- 展示 Accuracy、Precision、Recall、F1-score。
- 展示混淆矩阵，说明预测正确和错误的分布。
- 展示 loss 曲线，说明训练过程的变化。
- 页面还会给出测试样例预测结果。

当前项目中已有一份可展示的训练结果：

```text
storage/models/archive/task_2_20260415_222928/metrics.json
```

其中主要指标为：

```text
Accuracy: 0.828125
Precision: 0.891892
Recall: 0.825000
F1-score: 0.857143
```

可以这样说：

```text
评估页展示的是训练后模型在验证或测试数据上的效果。Accuracy 表示整体预测正确率，Precision 表示预测为积极的样本中有多少是真的积极，Recall 表示真实积极样本中有多少被找出来，F1 是 Precision 和 Recall 的综合指标。
```

### 第六步：历史记录

打开：

```text
http://127.0.0.1:8000/history
```

讲解重点：

- 单文本预测历史来自 `analysis_record`，记录来源为 `single`。
- 批量分析任务来自 `batch_task`，每个批量任务都可以点击“查看结果”查看明细。
- 批量分析明细也保存在 `analysis_record`，记录来源为 `batch`，并通过 `batch_task_id` 关联到批量任务。
- 页面支持删除选中记录和清空记录，这些操作会删除 SQLite 数据库中的数据。
- 训练任务来自 `train_task`，目前只查看，不提供清空入口。
- 这一步用来说明系统有数据持久化，不是刷新页面后结果就丢失。

可以这样说：

```text
历史记录页证明系统把核心操作都保存到了 SQLite。单文本记录和批量分析明细会区分来源保存，批量任务可以查看每一行的分析结果，也可以下载 CSV。删除和清空操作是真正删除数据库记录，所以页面会先弹出确认提示。
```

## 4. 记录管理和删除逻辑

答辩时如果老师问“历史记录怎么管理”，可以按下面说：

```text
系统把单文本分析和批量分析明细都保存在 analysis_record 表中，但通过 source_type 字段区分来源。单文本记录是 single，批量明细是 batch；批量明细还会通过 batch_task_id 关联到 batch_task 表。这样删除单文本记录时不会影响批量任务，删除批量任务时也能同步删除该批次对应的明细。
```

删除范围说明：

- 删除选中的单文本记录：只删选中的单文本数据库记录。
- 清空单文本记录：删除全部单文本数据库记录。
- 删除选中的批量任务：删除选中的批量任务和它们对应的批量明细。
- 清空批量任务：删除全部批量任务和全部批量明细。
- 导出的 CSV 文件默认保留在 `storage/exports/`，删除按钮只处理数据库记录。

## 5. 代码讲解主线

答辩时老师如果问代码，可以按这个顺序讲：

```text
run.py
  -> app/main.py
  -> app/routers/
  -> app/services/
  -> app/models/db/
  -> scripts/
```

对应说明：

- `run.py`：启动 Uvicorn 服务。
- `app/main.py`：创建 FastAPI 应用，注册页面和 API 路由，初始化数据库和模型。
- `app/routers/`：接收页面请求和 API 请求。
- `app/services/model_service.py`：负责模型加载和单文本预测。
- `app/services/batch_service.py`：负责 CSV 批量分析。
- `app/services/history_service.py`：负责历史记录查询、删除选中记录和清空记录。
- `app/services/train_service.py`：负责创建训练任务和启动训练脚本。
- `scripts/train_model.py`：真正执行模型训练、保存指标和模型文件。
- `scripts/evaluate_model.py`：离线评估当前激活模型。
- `scripts/batch_predict.py`：命令行批量分析入口。

## 7. 老师常问问题和回答

### 为什么选择 BERT？

BERT 是基于 Transformer 的预训练语言模型，适合处理中文语义理解任务。这里使用 `bert-base-chinese`，可以利用预训练模型已有的中文语言表示能力，再通过情感数据集进行二分类微调。

### 数据集是什么？

项目使用 ChnSentiCorp 中文情感分类数据集，标签是 `0=消极`、`1=积极`，适合酒店、商品、评论等中文情感判断场景。

### 系统怎么判断积极还是消极？

模型输出两个类别的 logits，经过 softmax 得到消极和积极概率。积极概率更高就输出“积极”，否则输出“消极”，置信度取两个概率中的最大值。

### 为什么使用 SQLite？

SQLite 不需要单独安装数据库服务，适合课程设计、毕业设计和本地答辩演示。它可以保存预测记录、批量任务、训练任务和模型注册信息。

### 为什么不现场完整训练？

BERT 训练对算力要求较高，CPU 环境下耗时较长。答辩现场更适合展示训练入口、训练参数、已有训练结果、评估指标和模型文件。如果老师要求现场演示，可以使用小参数跑一个短训练。

### 项目的完整性体现在哪里？

项目不是单独的模型脚本，而是包含 Web 页面、API、数据库、训练脚本、批量处理、模型评估和历史记录的完整系统。用户可以从页面完成预测和分析，开发者也可以通过脚本完成批量预测和评估。

### 批量分析后的结果为什么能重新查看？

批量任务会保存到 `batch_task` 表，批量中的每一条分析明细会保存到 `analysis_record` 表，并通过 `batch_task_id` 与批量任务关联。历史页点击“查看结果”时，会根据任务 ID 查询对应明细；如果是旧任务，也可以从导出的 CSV 文件读取预览。

### 删除记录会删除什么？

删除按钮会删除 SQLite 数据库中的记录。删除单文本记录只影响单文本分析历史；删除批量任务会同时删除批量任务和它对应的批量明细。默认不删除已经导出的 CSV 文件。
